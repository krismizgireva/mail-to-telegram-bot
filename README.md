# Yandex Mail → Telegram Bot

Готовый проект: читает непрочитанные письма из Яндекс Почты по IMAP и пересылает их в Telegram-чат через бота.

## Что делает
- Проверяет INBOX каждые 5 минут через GitHub Actions.
- Берет только непрочитанные письма.
- Пересылает в Telegram отправителя, тему, дату и начало текста письма.
- После успешной отправки помечает письмо как прочитанное.

## 1. Что нужно подготовить

### Telegram
1. Создайте бота через `@BotFather`.
2. Напишите боту любое сообщение.
3. Откройте в браузере:
   `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
4. Найдите `chat.id` — это значение сохраните.

### Яндекс Почта
1. В Яндекс Почте откройте настройки → «Почтовые программы».
2. Включите доступ через IMAP и опцию «Пароли приложений и OAuth-токены».
3. В Яндекс ID создайте пароль приложения типа «Почта».
4. Используйте этот пароль, а не обычный пароль от аккаунта.

Важно: по документации Яндекса пароль приложения может начать действовать через 2–3 часа.

## 2. Как запустить
1. Создайте новый приватный репозиторий на GitHub.
2. Загрузите в него файлы из этого архива.
3. Откройте `Settings` → `Secrets and variables` → `Actions`.
4. Создайте секреты:
   - `TELEGRAM_TOKEN`
   - `TELEGRAM_CHAT_ID`
   - `EMAIL_LOGIN`
   - `EMAIL_PASSWORD`
5. Перейдите во вкладку `Actions` и запустите workflow `Mail to Telegram` вручную через `Run workflow`.
6. После проверки оставьте расписание каждые 5 минут.

## 3. Какие значения ставить
- `EMAIL_LOGIN` — полный адрес ящика, например `you@yandex.ru`
- `EMAIL_PASSWORD` — пароль приложения Яндекса
- `TELEGRAM_TOKEN` — токен бота от BotFather
- `TELEGRAM_CHAT_ID` — ваш chat_id

## 4. Что можно быстро доработать
- Фильтр только по определенному отправителю: переменная `FROM_FILTER`
- Фильтр по теме: переменная `SUBJECT_FILTER`
- Отдельная папка вместо INBOX: переменная `IMAP_FOLDER`
- Более длинный текст письма: переменная `MAX_BODY`

## 5. Ограничения
- GitHub Actions по cron не гарантирует запуск секунда в секунду.
- Если в одном запуске много новых писем, сейчас отправляются последние 10.
- Вложения не пересылаются, только текст письма.

## 6. Безопасность
- Не храните токены и пароли в коде.
- Используйте только GitHub Secrets.
- Если пароль приложения скомпрометирован — удалите его в Яндекс ID и создайте новый.
